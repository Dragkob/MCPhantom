# Changelog

### Security

- Read-only scanning for mutating MCP tools - AutoPwn no longer fuzzes tools whose names or descriptions indicate write/store behavior (`store`, `write`, `save`, `upload`, `create`, `update`, `delete`, and similar). This prevents destructive side effects such as calling `store_file` with a traversal path in `file_name` and overwriting system files (e.g. `/etc/passwd`).
- Path traversal limited to read templates - Traversal checks run only against `resource_template` capabilities (e.g. `getfile://{file_name}`), not static `resource://` endpoints or MCP tools.
- Safer default tool filler values - Default path/file parameter values used when fuzzing other fields were changed from `/tmp/test.txt` to `test.txt` (mutating tools are now excluded entirely).

### Detection

- SQLi coverage for generic template parameters - Capabilities like `password://{platform}` are now classified and tested for SQL injection even when parameter names lack obvious SQL keywords.
  - Template placeholders are parsed and fuzzy-matched; unknown slots default to `DATABASE` / `SQLI`.
  - Description verbs (`fetch`, `stored`, `lookup`, `search`, etc.) imply database-backed lookups.
  - Expanded keyword map (`platform`, `name`, `search`, `filter`, `term`, and others).
  - AutoPwn safety net: any `resource_template` with `{...}` slots is eligible for SQLi testing.
- Path traversal evasion payloads - Added normalization-bypass payloads (`....//....//etc/passwd`, mixed separators, URL-encoding, semicolon bypass, Windows variants). Runtime limit raised to 18 payloads for the `TRAVERSAL` category.
- Improved traversal proof logic
  - False positives from error messages containing `/etc/passwd` as a path substring are rejected.
  - Linux disclosure requires passwd-style content (`root:x:`, `daemon:x:`, etc.), not path echoes alone.
  - Successful reads of sensitive targets (e.g. `/etc/passwd`) are reported even when file content is not classic passwd format (e.g. overwritten or unexpected body text).
  - Baseline comparison against a benign template read (`test`) to detect response changes.
- SQLi and traversal deduplication
  - Findings dedupe by response fingerprint, not exact payload string (collapses URL-encoded vs plain twins).
  - SQLi fingerprints normalize reflected URIs and `Unknown resource:` errors.
  - Traversal findings dedupe by response body; shorter payload kept when duplicates exist.
  - Duplicate finding events are suppressed during live AutoPwn streaming.
- Payload repository cleanup - SQLi payload list drops redundant URL-encoded twins when a plain-text equivalent exists.

### AutoPwn engine

- Dedicated traversal runner - `run_traversal_plugin()` executes payloads in parallel (with early stop on first confirmed hit) and uses a 90s check timeout.
- Template URI resolution aligned with manual reads - Resource template fuzzing uses the same `.format()`-based substitution as the dashboard (`audit/template_uri.py`), including placeholders like `{file_name*}`.
- Robust response parsing - Plugins use `response_text()` to fall back to `raw` MCP content when `response.text` is empty.
- Discovery hardening - Resource template URIs fall back across `uriTemplate`, `uri_template`, and `name` when fields are missing.
- Classifier - File-like URI schemes and templates (`getfile`, `filename`, `filepath`, etc.) explicitly tag `FILE` / `TRAVERSAL`.
- Job scheduling - File-like `resource_template` capabilities are always considered for scheduling even when tag overlap is thin.

### UI

- Action Result panel - Output is shown in a scrollable box instead of expanding unbounded.
- Findings evidence column - Full response text is displayed in a compact scrollable block; generic one-line messages like "Linux passwd-style file contents detected." were replaced with the actual response body.

### Fixed

- False positive path traversal on `getfile://{file_name*}` when the server returned errors containing `/etc/passwd` in the resolved path.
- Duplicate CRITICAL SQLi rows for the same capability caused by encoded/plain payload pairs and reflected-URI fingerprints.
- Duplicate traversal rows for different payloads that returned identical file contents.
- Path traversal checks timing out or reporting 0 hits when all payloads ran sequentially without early stop, or when detection only matched classic passwd formatting.
- Traversal not detected when manual reads succeeded but AutoPwn used different template expansion or empty `response.text`.
- Accidental file overwrite during scans via `store_file` and similar mutating tools (see Security).

---

### Notes for operators

- Restart the web server after pulling these changes so AutoPwn and the dashboard pick up new behavior.
- Static `resource://` endpoints may still receive SQLi heuristics in some builds; traversal and write-tool fuzzing are explicitly restricted as described above.
